## MUST-FIX: Negative number evasion (severity: critical, evasion rate: 50% in category)
#
# Root cause: _extract_dict_literal (line 252-268) and _track_subscript_assign
# (line 174-191) only handle ast.Constant values. On Python 3.13+, negative
# numbers are represented as ast.UnaryOp(USub, Constant(N)), not Constant(-N).
# Complex expressions like (1+0j) are ast.BinOp(Add, Constant(1), Constant(0j)).
# These value nodes are silently dropped, making the dict entry invisible.
#
# An attacker can evade detection with: subprocess.run(cmd, **{'shell': -1})

# Input 1: shell=-1 inline dict (truthy int, UnaryOp in AST)
import subprocess; subprocess.run(['ls'], **{'shell': -1})
# Expected: EXEC-002 (int -1 is truthy)
# Actual: NO FINDING (evasion)

# Input 2: shell=-1.0 inline dict (truthy float, UnaryOp in AST)

# Expected: EXEC-002 (float -1.0 is truthy)
# Actual: NO FINDING (evasion)

# Input 3: shell=(1+0j) inline dict (truthy complex, BinOp in AST)

# Expected: EXEC-002 (complex 1+0j is truthy)
# Actual: NO FINDING (evasion)
