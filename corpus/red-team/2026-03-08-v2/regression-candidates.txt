# ROOT CAUSE 1: Regex boundary off-by-one (CRITICAL / must-fix)
# The regex [A-Za-z0-9+/]{24,}={0,3} counts only charset chars, not padding.
# Payloads with 15-17 source bytes encode to 20-23 charset chars + padding,
# falling below the {24,} threshold. This silently drops ~25% of short PI
# phrases even when they ARE base64-encoded and properly quoted.
#
# Fix: Change {24,} to {20,} or count total length including padding, or
#      lower MIN_ENCODED_LENGTH. The regex and the Python length check
#      both need adjustment.
# =============================================================================
