# Python Repo Init Checklist

Blueprint for automating project scaffolding as a cookiecutter template or Claude skill.
Based on the skill-bank ‚Üí skill-scan setup pattern.

## Template Variables

| Variable | Example | Description |
|----------|---------|-------------|
| `project_name` | `skill-scan` | Repo/package display name (hyphenated) |
| `package_name` | `skill_scan` | Python package name (underscored) |
| `description` | `Security scanner for agent skills` | One-line project description |
| `python_version` | `3.13` | Target Python version |
| `dependencies` | `["click>=8.1.0,<9.0"]` | Runtime dependencies |
| `agent_dir` | `.agents` | Agent standards directory name |
| `repomap_path` | `../repomap` | Path to local repomap repo for install script |
| `source_repo` | `../skill-bank` | Path to repo to copy scripts/ and standards from |

## Steps (in order)

### 1. Project root files

| File | Templated? | Notes |
|------|-----------|-------|
| `pyproject.toml` | Yes | `project_name`, `package_name`, `description`, `python_version`, `dependencies`, build-system, tool configs (ruff, mypy, pytest) |
| `.python-version` | Yes | `python_version` |
| `.gitignore` | No | Standard across projects |
| `.env.example` | Minimal | Project-specific env vars |
| `Makefile` | Yes | `agent_dir` in EXCLUDE_DIRS and MYPY_EXCLUDE |
| `CLAUDE.md` | Yes | `project_name`, `package_name`, `description`, stack, structure |
| `README.md` | Yes | `project_name`, `description` |
| `.pre-commit-config.yaml` | No | Standard across projects (no `services/` dir) |

### 4. Scripts directory (copy from source repo)

Copy entire `scripts/` directory. These are project-agnostic quality tools:

- `__init__.py`
- `antipattern_context.py` ‚Äî context-aware checking
- `check_ast_antipatterns.py` ‚Äî AST antipattern detection
- `check_code_patterns.py` ‚Äî code pattern validation
- `check_float_equality.py` ‚Äî float equality in tests
- `check_test_patterns.py` ‚Äî test pattern validation
- `complexity/` ‚Äî file/function size and cyclomatic complexity
  - `__init__.py`, `__main__.py`, `analyzer.py`, `models.py`
- `arch_smell/` ‚Äî architecture smell detection
  - `__init__.py`, `analysis.py`, `audit.py`, `constants.py`, `git.py`, `infra.py`, `infra_calls.py`, `infra_patterns.py`, `models.py`

### 7. Install commands (post-scaffold)

```bash
# 1. Install dependencies
uv sync --group dev

# 2. Install repomap
bash {repomap_path}/deploy/install.sh .

# 3. Install pre-commit hooks
uv run pre-commit install

# 4. Generate initial repomap index
tools/repomap/repomap.sh index

# 5. Initial commit
git add -A && git commit -m "Initial project scaffold"
```

## File classification

### Fully reusable (copy as-is)
- `scripts/` (entire directory)
- `.agents/standards/code-rules.json`
- `.agents/standards/test-rules.json`
- `.agents/plans/templates/PLAN-TEMPLATE.yaml`
- `.agents/plans/templates/SPIKE-TEMPLATE.yaml`
- `.github/dependabot.yml`
- `.gitignore`
- `.pre-commit-config.yaml`
- `tests/constants.py`

### Needs template substitution
- `pyproject.toml` ‚Äî name, description, deps, python version, package path
- `Makefile` ‚Äî agent dir name
- `CLAUDE.md` ‚Äî name, description, stack, structure
- `.python-version` ‚Äî python version
- `.github/workflows/ci.yml` ‚Äî python version, agent dir name
- `.agents/WORKFLOW.md` ‚Äî project-specific rules
- `.agents/standards/CODE-PATTERNS.md` ‚Äî project-specific examples in patterns 1, 3, 4
- `.agents/standards/TEST-PATTERNS.md` ‚Äî project-specific examples in patterns 3, 4, 5

### Generated (not templated)
- `uv.lock` ‚Äî generated by `uv sync`
- `tools/repomap/` ‚Äî installed by repomap script
- `.repomap/` ‚Äî generated by repomap index
