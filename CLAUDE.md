# skill-scan

Security scanner for agent skills — detect prompt injection, malicious code, and data exfiltration before installation.

## Quick Reference

- **Plans & specs**: `.agent/plans/`
- **Code patterns and rules**: `.agent/standards/`
- **Workflow context**: `.agent/WORKFLOW.md`

## Stack

Python 3.13, click, ruff, mypy, pytest, bandit

## Workflow

1. Read existing code before modifying
2. Make changes
3. Run `make check`

```bash
make install  # Install dependencies
make check    # All quality checks (run early, run often)
```

## Code

- Follow `.agent/standards/CODE-PATTERNS.md` for design decisions
- Follow `.agent/standards/TEST-PATTERNS.md` when writing or modifying tests
- Rules enforced via `.agent/standards/code-rules.json`
- Type hints on all public functions
- Tests in `tests/`
- Core scanner engine and decoder use stdlib only (`re`, `pathlib`, `json`, `tomllib`, `base64`, `binascii`, `ast`, `concurrent.futures`, `urllib.parse`)
- Don't add deps without `uv` + `pyproject.toml`
- **Max 300 lines per file** (source code in `src/` and `tests/`)

## Architecture Notes

- `content_scanner.scan_all_files()` returns a 4-tuple: `(findings, bytes_scanned, files_skipped, suppressed_count)`
- Concurrent scanning uses `ProcessPoolExecutor` when file count >= `MIN_FILES_FOR_CONCURRENCY` (8); falls back to sequential on `OSError`/`RuntimeError`
- `ScanConfig.max_workers`: `0` = auto-detect (capped at 8), positive = explicit worker count (also capped at 8)
- `ScanResult.suppressed_count`: count of findings removed by inline `# noqa: RULE-ID` comments; default `0`
- Bare `# noqa` (no rule ID) does NOT suppress — security scanner requires explicit IDs
- `analyze_python()` returns an `AST-PARSE` INFO finding on `SyntaxError`/`ValueError`/`RecursionError` during parsing; returns an `AST-DEPTH` INFO finding (plus any accumulated findings) on `RecursionError` during tree walking
- `AST-PARSE` and `AST-DEPTH` findings are exempt from `active_ids` filtering in `content_scanner._apply_rules()` — they always propagate to output
- `MAX_AST_RESOLVE_DEPTH = 50` in `_ast_helpers.py` — recursive string-resolution helpers return `None` instead of crashing at depth > 50
- `match_content()` in `engine.py` is a public wrapper with no `_depth` parameter; `_match_content_recursive()` is the private implementation that carries `_depth`
- `ast_analyzer.py` and `decoder.py` are facade modules — they re-export all names from their sibling `_ast_detectors.py`/`_ast_rot13.py`/`_ast_symbol_table.py`/`_ast_split_detector.py` and `_decoder_helpers.py`/`_decoder_url_unicode.py` respectively (Facade Re-export Pattern); `_ast_split_resolve.py` re-exports `resolve_bytes_constructor` from `_ast_split_bytes.py`, `_resolve_reduce_concat` from `_ast_split_reduce.py`, `_resolve_join_call` from `_ast_split_join_helpers.py`, and `_resolve_format_call`/`_resolve_percent_format` from `_ast_split_helpers.py`
- `build_alias_map(tree)` in `_ast_helpers.py` returns `dict[str, str]` mapping local alias to canonical module name; called by `analyze_python()` and threaded to all `_detect_*` functions via `alias_map` kwarg
- `get_call_name(node, alias_map=None)` resolves aliased call names (e.g. `c.encode` with `alias_map={'c': 'codecs'}` → `'codecs.encode'`); all `_detect_*` functions accept `alias_map` kwarg (empty-dict default, backward-compatible)
- `ast_analyzer.py` uses a `_DETECTORS` tuple to register all detector functions; current detectors: `_detect_unsafe_calls`, `_detect_dynamic_imports`, `_detect_unsafe_deserialization`, `_detect_string_concat_evasion`, `_detect_dynamic_access`, `_detect_decorator_evasion`, `_detect_rot13_codec`, `_detect_rot13_maketrans`; add new detectors to this tuple
- `analyze_python()` also calls `build_symbol_table(tree)` and `detect_split_evasion(tree, file_path, alias_map, symbol_table)` separately from the `_DETECTORS` loop — tree-level detectors that need the full symbol table go here, not in `_DETECTORS`
- `build_symbol_table(tree: ast.Module) -> dict[str, str]` in `_ast_symbol_table.py` — two-pass pre-pass that returns a flat dict of variable-to-string mappings; Pass 1: variable assignment tracking (existing behavior); Pass 2: return-value extraction (stores converged function return values under `"funcname()"` / `"ClassName.method()"` composite keys) then call-site resolution (`_resolve_call_assignments` — resolves `x = func()` at module level when `"func()"` is tracked); function-scoped variables are prefixed `"funcname.varname"`; class attributes stored as `"ClassName.attr"` at module scope; bounded by `MAX_RESOLVE_DEPTH = 50`; circular references are dropped silently; tracks dict subscript assignments (`d['key'] = 'val'`) and list-index subscript assignments (`parts[0] = 'ev'`) as composite keys (`'varname[key]'`); tracks dict literal initializers and resolves `BinOp(Mult)` string repetition; handles `global`/`nonlocal` declarations via pre-pass (`_collect_scope_declarations`)
- `_ast_symbol_table_helpers.py` — private module with all assignment-tracking helpers used by `_collect_assignments()`; `_ast_symbol_table.py` imports these via a deferred local import inside `_collect_assignments()` to break the circular import that would otherwise result from `_ast_symbol_table_helpers.py` importing `_Ref` from `_ast_symbol_table.py`; also contains `_collect_scope_declarations(body)` and `_handle_self_attr_assign()` (delegates to `_ast_symbol_table_class_helpers.py`); `_sub_bodies()` was extracted to `_ast_symbol_table_return_helpers.py` (DRY consolidation — class_helpers imports it from there)
- Composite key format `'varname[key]'` (brackets) in the symbol table represents both dict subscript entries (`d['key'] = 'val'`) and list-index entries (`parts[0] = 'ev'`) — brackets prevent collision with plain variable names; integer index `0` and string key `'0'` produce the same composite key `'varname[0]'` (documented collision edge case, not guarded)
- `_resolve_binop_mult(node)` in `_ast_symbol_table_helpers.py` — resolves `string * positive-int` (and `int * string`) to the repeated string; only resolves when integer >= 1; skips 0, negative, and float operands
- `_collect_scope_declarations(body) -> tuple[set[str], set[str]]` in `_ast_symbol_table_helpers.py` — collects `global` and `nonlocal` declared names from a function body (walks immediate body plus if/else branches, does not recurse into nested functions); returns `(global_names, nonlocal_names)`
- `_route_globals(func_scope, global_names, result, module_scope=None)` in `_ast_symbol_table.py` — moves global-declared writes from function scope to the module-level result dict (last-write-wins); also updates `module_scope` so subsequent function scopes resolving against it see routed values; removes the function-prefixed key to prevent duplicates
- `_process_nested(body, parent_scope, result)` in `_ast_symbol_table.py` — handles nested functions: routes global-declared writes to module-level result dict and nonlocal-declared writes to the enclosing function scope; stores remaining inner-scope entries under `"funcname.varname"`
- `_process_class(node, module_scope, result)` in `_ast_symbol_table.py` — walks a `ClassDef` node: collects class-level assignments prefixed as `'ClassName.attr'`; then iterates methods and calls `_handle_self_attr_assign()` to capture `self.attr = 'val'` patterns also as `'ClassName.attr'`; also calls `_collect_return_value()` for each method and stores converged return values under `'ClassName.method()'` composite key
- `_resolve_call_assignments(body, result)` in `_ast_symbol_table.py` — post-pass over module-level `ast.Assign` nodes; when the RHS is an `ast.Call` and the call's composite key (e.g. `"func()"`) is already in `result`, stores the resolved value for the LHS variable; enables double-indirection patterns (`x = get_prefix(); result = x + suffix`)
- `_get_call_key(call) -> str | None` in `_ast_symbol_table.py` — maps a `Call` node to the composite key used for tracked return values: `"funcname()"` for plain calls, `"ClassName.method()"` for attribute calls; returns `None` for unrecognised patterns
- `detect_split_evasion(tree, file_path, alias_map, symbol_table) -> list[Finding]` in `_ast_split_detector.py` — reconstructs strings assembled via `BinOp(Add)`, f-string interpolation, `"".join(...)`, `'template'.format(...)`, `'%s%s' % (a, b)`, `.replace()` chains, `functools.reduce()` concat, dict subscript lookups, list-index subscripts, `ast.Attribute` (`self.attr`), `ast.Call` (function return values via `resolve_call_return()`), `reversed()` join, `chr()`/`ord()` chains, bytes-constructor patterns, comprehension `chr()` mapping, and dynamic dispatch via introspection subscripts using the symbol table; emits EXEC-002 for dangerous names (eval, exec, system, popen), EXEC-006 for dynamic import names (`__import__`, `getattr`); also bridges to decoder for split encoded payloads
- `_RESOLVERS` in `_ast_split_detector.py` — tuple of `(predicate, resolver)` pairs, modeled after `_DETECTORS` in `ast_analyzer.py`; `_try_resolve_split()` iterates the tuple and returns the first non-`None` result; predicates are short functions or lambdas checking `isinstance`; all resolvers share signature `(node, symbol_table, scope, *, alias_map=None) -> str | None`; add new string resolvers here, not in `_try_resolve_split()` directly
- `_resolve_join_call(node, symbol_table, scope, alias_map)` in `_ast_split_join_helpers.py` — dispatches to `_resolve_join_elements` (list/tuple), `_resolve_comprehension_join` (GeneratorExp or ListComp), `_resolve_reversed_join` (reversed()), or `_resolve_map_join` (map(chr/str)) depending on join argument type
- `resolve_fstring()` in `_ast_split_resolve.py` processes `ast.FormattedValue` by resolving `value.value` directly via `resolve_expr()`, ignoring `conversion` (!s/!r) and `format_spec` fields; evasion via `f'{x!r}'` is detected
- `_resolve_subscript_lookup(node, symbol_table, scope)` in `_ast_split_resolve.py` — resolves `ast.Subscript` (dict key or integer list index) to a string via composite key lookup; delegates to `_resolve_subscript_expr` in `_ast_split_helpers.py`; re-exported from `_ast_split_detector.py` (Facade Re-export Pattern)
- `resolve_expr(node, symbol_table, scope)` in `_ast_split_resolve.py` — resolves `ast.Name`, `ast.Attribute`, `ast.Subscript`, or `ast.Call` to a string; Attribute resolution is gated to `self`/`cls`.attr (looked up as `scope.attr`) and direct `ClassName.attr` patterns only — arbitrary `obj.attr` is rejected to prevent false positives; `ast.Call` nodes dispatch to `_resolve_chr_call()` first, then `resolve_bytes_constructor()`, then `resolve_call_return()`
- `resolve_call_return(node, symbol_table, scope)` in `_ast_split_resolve.py` — resolves an `ast.Call` node to a string via the parentheses-suffix composite key in the symbol table; handles plain calls (`func()` → looks up `"funcname()"`) and attribute calls with `self`/`cls` or direct `ClassName` receivers (`self.method()` → looks up `"scope.method()"`); returns `None` for unknown/untracked functions; re-exported from `_ast_split_detector.py`
- `resolve_binop_chain`, `resolve_operand`, `resolve_fstring` in `_ast_split_resolve.py` — expression resolution helpers; re-exported from `_ast_split_detector.py`
- `resolve_call(node, symbol_table, scope, *, alias_map=None)` in `_ast_split_resolve.py` — registry-compatible wrapper that chains all Call sub-resolvers: join, format, bytes-constructor, reduce, and call-return; registered in `_RESOLVERS` tuple; replaces the former `isinstance(node, ast.Call)` branch in `_try_resolve_split()`
- `resolve_percent_format(node, symbol_table, scope, *, alias_map=None)` in `_ast_split_resolve.py` — registry-compatible wrapper for `_resolve_percent_format`; registered in `_RESOLVERS` tuple for `BinOp(Mod)` nodes
- `_resolve_replace_chain(node, symbol_table, scope, *, alias_map=None)` in `_ast_split_resolve.py` — resolves chained `.replace(old, new)` calls; walks the chain up to 20 levels deep, resolves the base string from a literal or symbol table lookup, applies replacements left-to-right; registered in `_RESOLVERS` via `_is_replace` predicate
- `_resolve_chr_call(node)` in `_ast_split_chr.py` — resolves `chr(N)`, `chr(ord('x'))`, and `chr(ord('x') + N)` to a single character; dispatches to `_resolve_chr_arg` → `_resolve_int_arg`; `_resolve_int_arg` handles integer literals, `_resolve_ord_call` (single-char string constant), and `_resolve_int_binop` (Add/Sub/Mult on resolvable integer operands); returns `None` for non-constant arguments (R-IMP003); imported by `_ast_split_resolve.py`
- `resolve_bytes_constructor(node, alias_map)` in `_ast_split_bytes.py` — resolves bytes-constructor patterns to a decoded string; dispatches to `_resolve_bytearray_decode` (`bytearray(b'...').decode()`), `_resolve_str_bytes` (`str(b'...', 'utf-8')`), and `_resolve_codecs_decode` (`codecs.decode(b'...', 'utf-8')`); `_decode_bytes_encoding_args` shared helper extracts `(bytes_literal, encoding_str)` from two-arg calls; gates on literal bytes arguments only (R-IMP004); re-exported from `_ast_split_resolve.py` and `_ast_split_detector.py` (Facade Re-export Pattern)
- `_resolve_reduce_concat(node, symbol_table, scope, *, alias_map=None)` in `_ast_split_reduce.py` — resolves `functools.reduce(lambda a,b: a+b, [...])` and `functools.reduce(operator.add/concat, [...])` to concatenated strings; uses `alias_map` to resolve functools/operator aliases; gates on string-literal list/tuple as the iterable; re-exported from `_ast_split_resolve.py`
- `_NAME_RULE` in `_ast_split_detector.py` is a lookup table mapping each dangerous name to `(rule_id, severity, description_prefix)` — use this pattern when one detector must emit different rule IDs per matched name
- `_DECORATOR_RULE` in `_ast_detectors.py` — local dict mapping dangerous decorator names to `(rule_id, Severity, description_prefix)`; mirrors `_NAME_RULE` but defined locally to avoid the import cycle that would result from importing `_NAME_RULE` from `_ast_split_detector` (which already imports from `_ast_detectors`); used by `_detect_decorator_evasion`
- `_detect_decorator_evasion(node, file_path, *, alias_map=None)` in `_ast_detectors.py` — node-level detector for `@eval`, `@exec`, `@builtins.eval`, and other dangerous-name decorators on `FunctionDef`, `AsyncFunctionDef`, and `ClassDef` nodes; checks both `ast.Name` (plain decorator) and `ast.Attribute` (dotted decorator like `@builtins.eval`) forms; registered in `_DETECTORS` tuple in `ast_analyzer.py`; re-exported from `ast_analyzer.py`
- `_INTROSPECTION_FUNCS = frozenset({"vars", "globals", "locals"})` in `_ast_split_detector.py` — set of built-in introspection function names checked by `_is_introspection_base()`
- `_check_dynamic_dispatch(node, symbol_table, scope, file_path)` in `_ast_split_detector.py` — detects dynamic dispatch via `globals()['eval']`, `vars(obj)['eval']`, `obj.__dict__['eval']`, and two-level chaining (`globals()['__builtins__']['eval']`); called per-node before `_try_resolve_split`; gates on `_is_introspection_base()` plus key lookup in `_NAME_RULE`
- `_is_introspection_base(node)` in `_ast_split_detector.py` — returns `True` if node is `ast.Call` to a name in `_INTROSPECTION_FUNCS` or an `ast.Attribute` with `attr == '__dict__'`
- `_extract_subscript_key(slice_node, symbol_table, scope)` in `_ast_split_detector.py` — extracts subscript key as string from `ast.Constant` (string) or `ast.Name` (via `_scoped_lookup`); returns `None` for non-resolvable slices
- `_resolve_reversed_join(call, sep, symbol_table, scope)` in `_ast_split_join_helpers.py` — resolves `reversed()` as a join argument; gates to string literals, List/Tuple of tracked elements, and tracked variable names; dispatches to `_resolve_reversed_inner` to get the list of parts, then reverses and joins
- `_resolve_reversed_inner(inner, sep, symbol_table, scope)` in `_ast_split_join_helpers.py` — resolves the inner argument of `reversed()` to a list of characters/parts; handles `str` literal (returns character list), `List`/`Tuple` (joins via `_resolve_join_elements` then splits), and `ast.Name` (symbol table lookup then character list)
- `_ast_split_helpers.py` — private module with format/%-format resolution helpers; `_resolve_format_call(node, symbol_table, scope)` handles `'template'.format(a, b)` (gates on string-constant receiver); `_resolve_percent_format(node, symbol_table, scope)` handles `'%s%s' % (a, b)` (gates on string-constant LHS, avoids integer modulo); `_PERCENT_SPEC_RE` matches all standard %-specifiers (`%s %d %f %r %x %o %e %g %c %a %i`) with `(?<!%)` lookbehind to exclude `%%`; `_substitute_percent()` returns `None` when `len(values) > placeholder count` (over-provisioning defense); `_scoped_lookup`, `_resolve_join_elements`, `_resolve_subscript_expr`, `_resolve_subscript_key`, and `_resolve_single_expr` live here; `_resolve_single_expr` gates `ast.Attribute` resolution to `self`/`cls`.attr and `ClassName.attr` patterns only (prevents false positives on arbitrary `obj.attr`); `_resolve_subscript_key(slice_val)` converts string key or non-negative int to composite key suffix; `_resolve_subscript_expr` now handles `ast.Slice` nodes via `_resolve_slice_expr` (dispatched before integer-index handling); `_extract_slice_int(node) -> tuple[bool, int | None]` extracts an integer from an optional slice bound (open-ended = `None`; non-Constant bound returns `ok=False`); `_resolve_slice_expr(var_name, slc, symbol_table, scope)` looks up the variable in the symbol table and applies Python slicing — requires Constant start/stop bounds; negative step is rejected; re-exports `_resolve_generator_join` and `_resolve_map_join` from `_ast_split_join_helpers.py`
- `_ast_split_join_helpers.py` — private module with generator expression, list comprehension, and map() resolution helpers; `_resolve_join_call(node, symbol_table, scope, alias_map)` is the top-level join dispatcher (moved here from `_ast_split_detector.py`); `_resolve_comprehension_join(node, sep, symbol_table, scope)` handles both `GeneratorExp` and `ListComp` — identity patterns (`x for x in [...]`) and chr()-mapping patterns (`chr(c) for c in [ints]`); `_resolve_generator_join` is a thin shim that delegates to `_resolve_comprehension_join`; `_resolve_comprehension_chr(elt, target_id, iter_elts, sep)` handles inline integer list chr() mapping; `_resolve_map_join(call, sep, alias_map)` dispatches to `_resolve_map_chr` (int literals → characters) or `_resolve_map_str` (string passthrough); `map()` support is limited to `chr()` and `str()` only
- `_Ref` sentinel class in `_ast_symbol_table.py` marks unresolved variable references during the pre-pass; resolved to `str` or dropped before `build_symbol_table()` returns
- `_ast_symbol_table_return_helpers.py` — private module with return-value extraction logic; `_collect_return_value(func_body, scope) -> str | None` returns the converged string when ALL return paths resolve to the same constant, or `None` otherwise (conservative); checks for implicit fallthrough (paths with no return) via `_has_implicit_fallthrough()`/`_definitely_returns()`; handles `if/elif/else`, `try/except/else/finally`, `with`, and `match` blocks; skips nested function/class defs during the return walk; resolves `ast.Constant`, `ast.Name` (via scope lookup), and `ast.BinOp(Add)` in return expressions; also owns `_sub_bodies()` (DRY consolidation — previously duplicated in `_ast_symbol_table_class_helpers.py`); `_ast_symbol_table.py` imports via deferred local import inside `build_symbol_table()` and `_process_nested()` to avoid circular imports
- Known limitations of return-value tracking: only resolves `ast.Constant`, `ast.Name` (scope lookup), and `ast.BinOp(Add)` within return expressions — complex returns (calls, subscripts, f-strings) are not tracked (conservative); method returning a class-level variable via `self.attr` is NOT tracked by `_collect_return_value` (conservative gap); `_resolve_call_assignments` handles module-level `x = func()` only — function-body-level call-site assignment is resolved inline by the split detector via `resolve_call_return()`; only same-module calls are resolved — no cross-module or dynamic dispatch
- Known deferred evasion patterns (debt): DEBT-024-KWARGS-UNPACKING — star-args/kwargs unpacking detection not yet implemented; DEBT-024-TRACKED-COMPREHENSION — `''.join(chr(c) for c in codes)` where `codes` is a tracked variable requires symbol table extension to support non-string values (currently dict[str, str] cannot hold integer lists)
